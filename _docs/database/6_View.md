View는 가상 테이블로, 사용자에게 **접근이 허용된 자료만을 제한적**으로 보여주기 위해 하나 이상의 테이블로 부터 유도된 이름을 가진 테이블을 의미한다.

![img](https://i.loli.net/2020/11/28/pjv2XWTAqZQbmPM.png)



## View의 특징

1. View는 가상 테이블이므로 물리적으로 구현되지는 않는다.
   * 대신, View에 대한 정의만 저장된다.
2. 정의된 view는 다른 뷰의를 정의할 때 사용할 수 있다.
   * view가 정의된 기본 테이블이나 view를 삭제하면, 그 테이블이나 view를 기초로 정의된 다른 view도 자동으로 삭제된다.

3. **데이터에 대한 논리적 독립성을 제공한다.**
   * 데이터 독립성: 데이터의 저장 구조나 처리 방식이 변경되어도 응용 프로그램에 영향을 주지 않도록 하는 특성
     1. 논리적 데이터 독립성: 각 응용 프로그램들이 서로 영향을 받지 않으면서 응용 프로그램이 원하는 논리적 구조를 제공할 수 있는 특성
     2. 물리적 데이터 독립성: 응용 프로그램에 영향을 미치지 않고 데이터의 물리적 구조를 변경할 수 있는 특성
4. View로 정의된 내용은 갱신, 삽입, 삭제 등과 같은 연산에 대해 많은 제약이 따른다.
   * `ALTER VIEW`와 같은 View 정의에 대한 수정은 금지되어 있다.
   * View는 자신을 생성하기 위해 이용된 테이블의 기본 키를 포함하고 있어야 연산이 가능하다.
5. View는 복잡한 질의를 단순화 하는데 도움을 준다.
6. 접근 제어를 통한 보안 기능을 제공한다
   * 특정 필드만을 선택하여 뷰를 생성할 수 있기 때문에 응용 프로그램은 선택되지 않은 필드의 존재를 알 수 없으며, 접근 또한 할 수 없다.   
      :arrow_right: 따라서, 뷰를 이용하면 특정 데이터에 대한 접근 제어와 보안 기능을 제공할 수 있다.
